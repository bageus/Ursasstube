# Техническое ревью и рекомендации по улучшению

## Что проверено
- Структура проекта и входная HTML-разметка.
- Основные клиентские модули (`js/config.js`, `js/api.js`, `js/auth.js`, `js/ui.js`, `js/store.js`).
- Базовые риски по поддерживаемости, производительности и безопасности.

## Краткий итог
Проект выглядит рабочим и уже содержит важные улучшения по XSS (см. `SECURITY_REVIEW.md`). При этом есть несколько зон, которые дадут заметный прирост стабильности и упростят дальнейшую разработку.

## Рекомендации (по приоритету)

### P0 (сделать в первую очередь) 
1. **Убрать inline-обработчики из HTML (`onclick`) и перейти на `addEventListener`.** //выполнено
   - Это нужно для безопасного внедрения строгой CSP без `unsafe-inline`.
   - Параллельно упростит тестирование UI и снизит связность между HTML и JS.

2. **Добавить SRI (`integrity`) для CDN-скриптов или перенести критичные зависимости локально.**
   - Сейчас внешние зависимости грузятся напрямую из CDN, что увеличивает supply-chain риск.

3. **Централизовать обработку сетевых ошибок и таймаутов для `fetch`.**
   - Сейчас запросы разбросаны по модулям; полезно сделать единый `request()` helper с `AbortController`, ретраями и унифицированным форматом ошибок.

### P1 (следующий шаг)
4. **Вынести runtime-конфигурацию в отдельный env/config слой.**
   - `BACKEND_URL` и др. параметры лучше передавать через окружение/файл конфигурации для dev/stage/prod.

5. **Снизить “шум” логов в production.**
   - Ввести уровни логирования (`debug/info/warn/error`) и переключение по флагу.

6. **Добавить базовый CI для статических проверок.**
   - Минимум: HTML-валидация, линтер JS, форматирование и smoke-check загрузки страницы.

### P2 (улучшения качества)
7. **Постепенно типизировать код (JSDoc/TypeScript).**
   - Особенно для payload-ов API и игровых state-структур.

8. **Добавить простые e2e smoke-тесты.**
   - Сценарии: старт игры, загрузка таблицы лидеров, открытие/закрытие Store, обработка ошибок API.

## Быстрые win-и (можно сделать за 1-2 итерации)
- Внедрить `apiClient`-обёртку над `fetch`.
- Перенести все `onclick` в JS-инициализацию.
- Подготовить CSP в режиме Report-Only и собрать отчёт нарушений за 3–7 дней.
- Добавить workflow в GitHub Actions: lint + smoke.

## Исправление в рамках этого ревью
