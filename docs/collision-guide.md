# Collision guide

## Короткий ответ

Столкновение в игре считается **не по пикселям спрайта**, а по двум логическим проверкам:

1. объект попал в нужное окно глубины `z`;
2. объект совпал с игроком по нужному правилу полосы (`lane`) или специальному правилу (spin / magnet).

Именно поэтому возможны оба эффекта:

- **"До объекта ещё визуально далеко, а hit уже случился"**;
- **"Я уже почти на линии объекта, а hit ещё не случился"**.

## Какие числа реально используются

В конфиге сейчас:

- `CONFIG.PLAYER_Z = 0.3`;
- `CONFIG.TUBE_Z_STEP = 0.086`.

Базовое окно проверки считается так:

- `collisionDepthMin = 0.3 + 0.086 = 0.386`;
- `collisionDepthMax = 0.3 + 0.086 * 2 = 0.472`.

От него строятся реальные окна столкновений:

| Тип | Формула | Реальный диапазон `z` |
|---|---|---|
| Obstacle | `base ± 0.2 * step` | **0.3688 .. 0.4892** |
| Bonus | `base ± 0.35 * step` | **0.3559 .. 0.5021** |
| Coin (lane) | `base ± 0.45 * step` | **0.3473 .. 0.5107** |
| Coin (spin) | `base ± 0.6 * step` | **0.3344 .. 0.5236** |
| Perfect Spin ring | `PLAYER_Z ± 0.4` | **-0.1 .. 0.7** |

## Почему столкновение может ощущаться "раньше"

Потому что препятствие не обязано доехать прямо до `PLAYER_Z = 0.3`. Для obstacle hit срабатывает уже когда объект входит в диапазон **0.3688 .. 0.4892**, то есть примерно **на 1–2 логических шага трубы впереди игрока**.

Проще говоря:

```text
игрок стоит здесь:                 z = 0.3000
игра уже проверяет obstacle hit:   z = 0.3688 .. 0.4892
```

На экране спрайт в этот момент ещё может казаться "не вплотную", потому что рендер использует перспективное масштабирование по `z`, а не физический хитбокс по пикселям.

## Почему объект может быть "на моей линии", но столкновения нет

Потому что при перестроении есть **визуальная полоса** и **логическая полоса**:

- визуально персонаж плавно едет в `targetLane`;
- логически `player.lane` остаётся старым до конца transition.

То есть ситуация выглядит так:

```text
Кадр 1:  player.lane = 0, targetLane = +1
Кадр 2:  спрайт уже между 0 и +1
Кадр 3:  спрайт почти на +1
Кадр 6:  только здесь player.lane становится +1
```

Пока transition не закончился, obstacle/bonus/обычная coin всё ещё сверяются со **старым** `player.lane`.

## Правила по типам объектов

### 1) Препятствия

Для препятствия одновременно нужно:

- `o.lane === player.lane`;
- `o.z` попал в `0.3688 .. 0.4892`.

Если щита нет — ран заканчивается.

### 2) Бонусы

Для бонусов почти то же самое:

- `b.lane === player.lane`;
- `b.z` попал в `0.3559 .. 0.5021`.

Поэтому бонус иногда подбирается чуть раньше, чем obstacle на похожей позиции.

### 3) Обычные монеты на полосе

Для обычной lane-монеты нужно:

- `c.lane === player.lane`;
- `c.z` попал в `0.3473 .. 0.5107`.

То есть окно у монеты ещё чуть шире, чем у препятствия.

### 4) Монеты кольца и spinOnly

Если монета `spinOnly`, то она собирается **только** если `player.isSpin === true` и `c.z` в диапазоне `0.3344 .. 0.5236`.

Если монета относится к `isCircle` / `isCircleInner`, её тоже можно собрать спином в этом более широком окне.

### 5) Магнит

Магнит вообще использует другой принцип:

- монета должна быть в `z` примерно между `0.05` и `1.5`;
- затем игра считает **экранную дистанцию** между монетой и игроком;
- если дистанция меньше радиуса магнита, монета собирается даже без совпадения полосы.

Поэтому с магнитом монета может улететь к игроку "не с моей дорожки" — и это нормально для текущей логики.

## Самые частые игровые ощущения

### Сценарий A — "слишком ранний obstacle hit"

Ты видишь большой спрайт перед собой и кажется, что место ещё есть. Но игра уже считает, что объект попал в рабочее окно `z`, и этого достаточно для столкновения.

### Сценарий B — "я уже визуально перестроился"

На экране медведь уже почти в новой полосе, но код ещё сравнивает объект со старым `player.lane`. Поэтому иногда hit не происходит, хотя картинка подсказывает обратное.

### Сценарий C — "монета рядом, но не берётся"

Обычно причина одна из трёх:

- это не ваш логический `lane`;
- монета ещё не вошла в нужное окно `z`;
- это ring/spin монета, а спин не активирован.

## Как читать схему

В файле `docs/collision-diagram.svg` показано сразу три вещи:

- шкала `z` с **реальными числовыми окнами**;
- различие между **визуальным** и **логическим** перестроением;
- разные правила для `obstacle`, `bonus`, `coin`, `spin`, `magnet`.


## Где крутить момент столкновения в коде

Если хочешь экспериментировать с «идеальным» моментом столкновения, смотри в первую очередь сюда:

### 1) Базовая глубина игрока

**Файл:** `js/config.js`

- `CONFIG.PLAYER_Z` — базовая логическая глубина игрока;
- `CONFIG.TUBE_Z_STEP` — шаг глубины между «ячейками» трубы.

Именно эти два параметра задают базу для всех окон столкновений.

### 2) Базовые окна по z для obstacle / bonus / coin

**Файл:** `js/physics.js`

Главный блок, который надо крутить:

```js
const collisionDepthMin = CONFIG.PLAYER_Z + CONFIG.TUBE_Z_STEP;
const collisionDepthMax = CONFIG.PLAYER_Z + CONFIG.TUBE_Z_STEP * 2;

const obstacleCollisionMin = collisionDepthMin - CONFIG.TUBE_Z_STEP * 0.2;
const obstacleCollisionMax = collisionDepthMax + CONFIG.TUBE_Z_STEP * 0.2;
const bonusCollisionMin = collisionDepthMin - CONFIG.TUBE_Z_STEP * 0.35;
const bonusCollisionMax = collisionDepthMax + CONFIG.TUBE_Z_STEP * 0.35;
const coinSpinCollisionMin = collisionDepthMin - CONFIG.TUBE_Z_STEP * 0.6;
const coinSpinCollisionMax = collisionDepthMax + CONFIG.TUBE_Z_STEP * 0.6;
const coinLaneCollisionMin = collisionDepthMin - CONFIG.TUBE_Z_STEP * 0.45;
const coinLaneCollisionMax = collisionDepthMax + CONFIG.TUBE_Z_STEP * 0.45;
```

Что менять:

- хочешь **позже obstacle hit** → уменьши `collisionDepthMin/Max` или уменьши коэффициенты `0.2`;
- хочешь **раньше obstacle hit** → увеличь `collisionDepthMin/Max` или увеличь коэффициенты `0.2`;
- хочешь **более щадящий подбор бонусов/монет** → расширяй коэффициенты `0.35`, `0.45`, `0.6`;
- хочешь **более точный, требовательный hit** → сужай эти коэффициенты.

### 3) Проверка Perfect Spin для кольцевых монет

**Файл:** `js/input.js`

Тут отдельное окно:

```js
if (c.isCircle && c.z >= CONFIG.PLAYER_Z - 0.4 && c.z <= CONFIG.PLAYER_Z + 0.4)
```

Если хочешь, чтобы Perfect Spin срабатывал:

- **реже / точнее** → уменьши `0.4`;
- **раньше и мягче** → увеличь `0.4`.

### 4) Магнит

**Файл:** `js/physics.js`

Тут важны две вещи:

```js
if (magnetActive && playerPos && c.z > 0.05 && c.z < 1.5)
const magnetRangeSq = 150 * 150;
```

Что менять:

- `0.05 .. 1.5` — диапазон по глубине, где магнит вообще работает;
- `150 * 150` — радиус магнита на экране.

Если монеты «слишком рано летят» к игроку, сначала уменьшай `150`, а потом уже сужай диапазон `z`.

### 5) Визуальное перестроение против логического lane

**Файл:** `js/physics.js`

Логический `player.lane` меняется только здесь, в конце transition:

```js
if (player.laneAnimFrame >= CONFIG.LANE_TRANSITION_FRAMES) {
  player.lane = player.targetLane;
}
```

А визуально игрок начинает ехать в новую полосу раньше.

Если хочешь, чтобы ощущение столкновения было «честнее» визуально, можно экспериментировать с:

- `CONFIG.LANE_TRANSITION_FRAMES` в `js/config.js`;
- моментом, когда `player.lane` переключается на `targetLane`.

### Моя рекомендация для первого этапа тюнинга

Я бы крутил параметры в таком порядке:

1. `obstacleCollisionMin / obstacleCollisionMax`;
2. `bonusCollisionMin / bonusCollisionMax`;
3. `coinLaneCollisionMin / coinLaneCollisionMax`;
4. `coinSpinCollisionMin / coinSpinCollisionMax`;
5. `CONFIG.LANE_TRANSITION_FRAMES`;
6. `Perfect Spin ±0.4`;
7. магнит `150 * 150`.

Так ты сначала поймаешь **ощущение честного obstacle hit**, а уже потом подгонишь более «приятные» бонусы и монеты.


## Готовые пресеты для быстрого переключения

Я вынес быстрые пресеты прямо в код:

- `arcade` — столкновения и подбор ощущаются мягче и чуть раньше;
- `fair` — текущее поведение по умолчанию, близкое к существующему балансу;
- `strict` — окна уже, магнит слабее, подбор и hit ощущаются строже.

Смотри файл `js/config.js`:

- `COLLISION_PRESETS`
- `ACTIVE_COLLISION_PRESET`
- `COLLISION_TUNING`

Чтобы переключить режим, достаточно поменять:

```js
const ACTIVE_COLLISION_PRESET = 'fair';
```

например на:

```js
const ACTIVE_COLLISION_PRESET = 'arcade';
```

или:

```js
const ACTIVE_COLLISION_PRESET = 'strict';
```
