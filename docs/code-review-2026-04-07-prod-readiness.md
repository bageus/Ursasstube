# Код-ревью перед запуском в прод (2026-04-07)

## Что проверено

- Полный проектный пайплайн: `npm run check`.

## Результаты

### 1) Ошибки/стабильность

- Критичных ошибок сборки, синтаксиса и статанализа не обнаружено.
- Все встроенные проверки из `npm run check` прошли успешно.
- Тесты (`node --test scripts/request.test.mjs`) прошли: 4/4.

### 2) Дубли

- По дереву runtime-зависимостей дубли пакетов не выявлены.
- В runtime используется один локальный пакет: `@walletconnect/ethereum-provider`.

### 3) Уязвимости

- Автоматический аудит через npm registry не выполнен из-за сетевого запрета/403:
  - `POST https://registry.npmjs.org/-/npm/v1/security/advisories/bulk -> 403 Forbidden`.
- Из-за этого нельзя подтвердить актуальный CVE-статус через `npm audit` в текущем окружении.

## Риски перед продом

- Основной блокер именно по security-проверке: отсутствует подтверждённый результат `npm audit`.
- Дополнительный сигнал: в окружении есть предупреждение npm `Unknown env config "http-proxy"` (не падение, но стоит очистить/исправить конфиг CI).

## Рекомендации

1. Прогнать в CI/CD или локально с доступом к npm audit endpoint:
   - `npm audit --omit=dev --audit-level=moderate`.
2. Зафиксировать в release-checklist обязательный security gate перед деплоем.
3. Убрать/исправить некорректный npm config `http-proxy`, чтобы исключить шум в логах и будущие регрессии совместимости npm.
4. Для более глубокой проверки дубликатов в кодовой базе (не только dependency tree) добавить отдельный инструмент в CI (например, jscpd) с baseline-порогом.

## Вывод

- По качеству кода и стабильности текущий проект выглядит готовым к прод-запуску.
- По безопасности запуск **условно готов**, но нужен успешный `npm audit` в среде с доступом к registry advisories.
